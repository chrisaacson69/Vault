# Connectors — External Communication Channels

Decision framework and registry for connecting the vault's skill system to external communication channels. Connectors allow skills to push updates, reports, and notifications to people outside the vault.

## Message Priority Framework

Connectors send to humans. Humans have limited attention. Every outbound message must earn its send.

### Priority Levels

| Priority | When to Send | Frequency Limit | Examples |
|---|---|---|---|
| **P1 — Action needed** | Recipient needs to do something | As needed | Interview scheduled, offer received, document needs signature |
| **P2 — Decision point** | A finding changes the strategy | 1-2 per week max | Strong posting match found, salary data significantly different from expectations, red flag on a target company |
| **P3 — Progress update** | Meaningful milestone reached | Weekly digest | "Applied to 3 roles this week, 1 interview scheduled, salary research complete for Denver market" |
| **P4 — FYI** | Interesting but not urgent | Only on request | New tool built, vault page created, industry trend noted |

### Rules

1. **Default to P3 weekly digest** — batch updates into a single weekly message unless something is P1/P2
2. **Never auto-send P4** — if they want it, they'll ask
3. **P1 is the only interrupt** — everything else can wait for the digest
4. **Include context, not detail** — "Found a strong match: Director of Agentic AI at Experian, $153K-$275K remote. Full briefing ready." Not the full briefing itself.
5. **Always include a next step** — what should the recipient do with this information?
6. **The human approves the first send** — Chris confirms the channel works and the tone is right before any automation

### The Digest Pattern

Weekly career coach update (P3):
```
Career Update — Week of [date]

Search: [N] postings evaluated, [M] rated A/B
Applications: [list if any]
Interviews: [list if any]  
Key finding: [one sentence — the most important thing this week]

Next step: [what you're doing next / what you need from them]
```

This pattern keeps coaches informed without requiring them to parse vault-level detail. They get the signal, not the noise.

## Design Principles

### Transparency

Connectors should be **transparent to the skill** — a skill says "notify the career coach about this finding" and the connector handles the how. The skill doesn't need to know whether it's WhatsApp, email, or carrier pigeon.

This means connectors sit at the **handler level** in the shared library:

```
Skill: "send this salary report to the career coach"
    │
    ▼
Dispatcher: notify.md → "who is the recipient? what channel?"
    │
    ▼
Handler: whatsapp.md / email.md / slack.md → "format and send"
    │
    ▼
Specific tool: WhatsApp API / Gmail MCP / Slack MCP
```

### When to Formalize

A connector earns formalization when:
1. **You have the access** — API key, phone number, auth token
2. **The pattern repeats** — sending the same type of update more than 2-3 times manually
3. **The format stabilizes** — you know what a "career coach update" looks like

Until then, it's fine to send manually and let the pattern emerge.

### Message Formatting

When a connector sends content generated by a skill, it should:
- **Summarize, don't dump** — a WhatsApp message is not a vault page
- **Include the source** — "Based on salary-intel analysis of Director of AI roles..."
- **Link back** — include a URL to the full report if the channel supports it
- **Match the channel's norms** — WhatsApp is informal, email is structured, Slack is threaded

### Privacy and Scope

- **Never send vault content that isn't explicitly marked for sharing**
- **Always confirm before first send to a new channel** — the user approves the channel, then subsequent sends can be automatic
- **Log what was sent** — keep a record of outbound messages for audit

## WhatsApp Integration Notes

When the WhatsApp number is available, evaluate these options:

| Option | Complexity | Capability |
|---|---|---|
| **WhatsApp Business API** (via Twilio/MessageBird) | MCP server wrapping REST API | Full automation, templates, media |
| **WhatsApp Web automation** (via Playwright) | Heavy, fragile | Screen automation, no official API |
| **Manual with pre-formatted messages** | Zero setup | Claude generates the message, user sends it | 

Start with option 3 (Claude formats, user sends). Graduate to option 1 when the volume justifies the setup.
